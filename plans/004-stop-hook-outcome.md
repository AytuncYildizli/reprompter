## Why this matters

The ambient gate (v12.8.0) closed the FRONT of the learning loop: every prompt is scored and weak ones get a nudge, with privacy-safe local events. The BACK is still open: nobody knows whether nudges are *accepted* (user says "reprompt this") or *ignored*. That acceptance signal is exactly what the deferred heuristic-recalibration task needs — without it, threshold tuning stays guesswork. A Claude Code `Stop` hook can passively answer "was this session nudged, and did a reprompter invocation follow?" and record ONE boolean-only event per session. Same design contract as the gate: fail-soft, exit 0 always, zero network, no prompt text persisted.

## Current state

- `scripts/prompt-gate.js` — the gate. Key reusable internals: `cacheDir(env)` / `defaultStatePath(env)` (XDG cache resolution), `hashedSessionId(sessionId)` (sha256 12-hex), `emitTelemetry` pattern (lazy `require("./telemetry-store")`, `dirPath: path.join(cacheDir(env), "telemetry")`, try/catch fail-soft). It records nudges as events: `{runId: "gate-<hash>", taskId: "ambient-gate", stage: "gate_prompt", metadata: {nudged: true|false, reason, ...}}` in `<cache>/reprompter/telemetry/events.ndjson`.
- `scripts/telemetry-schema.js` — `STAGES` set; you will add `"gate_outcome"` (one line, next to `"gate_prompt"`).
- `scripts/telemetry-store.js` — `createTelemetryStore({dirPath}).readEvents({limit})` and `writeEvent(...)` already exist; reuse, do not reimplement.
- `plugin/hooks/hooks.json` is GENERATED by `scripts/package-plugin.js` — the hooks content lives in the generator. You will extend the generator so the plugin ships BOTH hooks (UserPromptSubmit → prompt-gate.js, Stop → stop-gate.js).
- `scripts/plugin-package.test.js` — asserts hook command file-existence by replacing `${CLAUDE_PLUGIN_ROOT}`; extend for the second hook.
- Claude Code `Stop` hook contract: fires when the main agent finishes responding (can fire multiple times per session); stdin JSON includes `session_id`, `transcript_path` (JSONL file of the conversation), `hook_event_name: "Stop"`, `stop_hook_active`. Exit 0 stdout is NOT injected as context for Stop hooks (only UserPromptSubmit does that) — this hook only observes and records. It must NEVER exit 2 (that would block stopping).
- Transcript JSONL lines are objects with varying shapes; reprompter invocations appear as Skill/tool-use entries whose content mentions the skill name `reprompter`, and user acceptance often as a user message containing the word `reprompt`. Detection must be substring/boolean only — the transcript is read locally and ONLY booleans ever leave the function.
- Test conventions: `node:test`, `assert/strict`, `mkdtempSync`, spawned-CLI checks with `XDG_CACHE_HOME` pointed at temp dirs, env pinned (see `scripts/prompt-gate.test.js` `runGate` — after the v12.8.0 env-leak fix, ALWAYS pin gate env vars in spawned children).

## Steps

### Step 1: `scripts/stop-gate.js`

Header comment mirrors prompt-gate.js design rules (FAIL-SOFT / OPT-IN / PRIVACY / NEVER-BLOCK — a Stop hook must never exit 2). Reuse via `require("./prompt-gate")` any exported helpers; if `cacheDir`/`hashedSessionId` are not exported from prompt-gate.js, EXPORT them there (additive export only — do not change behavior) rather than duplicating.

Exports:
1. `detectNudge(events, hashedId)` — given telemetry events array, true iff a `gate_prompt` event with `runId === "gate-"+hashedId` and `metadata.nudged === true` exists.
2. `detectAcceptance(transcriptText, afterIso?)` — boolean: transcript contains a reprompter invocation signal (case-insensitive substring `"reprompter"` in a tool/skill-use line OR `"reprompt this"` anywhere). Pure function on text; returns boolean only.
3. `alreadyRecorded(events, hashedId)` — true iff a `gate_outcome` event for that hash exists (dedupe: Stop fires multiple times).
4. Hook CLI (`require.main`): read stdin JSON (`session_id`, `transcript_path`); fail-soft empty exit 0 on anything malformed. If `env.REPROMPTER_TELEMETRY === "0"` or `env.REPROMPTER_AMBIENT === "0"` → exit 0 silently. Read the telemetry store (same dirPath as the gate); if not nudged or already recorded → exit 0. Else read `transcript_path` (try/catch; missing file → exit 0), compute acceptance, write ONE event: `{runId: "gate-"+hash, taskId: "ambient-gate", stage: "gate_outcome", status: "ok", metadata: {accepted: <bool>}}`. Print nothing ever. Exit 0 always.

**Verify**: `echo 'garbage' | node scripts/stop-gate.js; echo $?` → no output, 0.

## Maintenance notes

- This is nudge-acceptance v1, NOT full artifact-outcome recording — `outcome-record.js` integration (joining gate sessions to flywheel records) is the follow-up once acceptance data proves the pipeline.
